// capabilities

Everything You Need.
Nothing You Don't.

A complete media processing toolkit for creators, researchers, and audio professionals.

🧠
Ultimate Vocal Isolation
Meta's htdemucs_ft with --shifts=2 (normal) or --shifts=4 --overlap=0.25 (extreme mode) and --two-stems=vocals. The most accurate vocal separation configuration available — zero music bleed in the output stem.
Zero Music Bleed
🌐
Universal Downloader
Powered by yt-dlp — download audio or video from YouTube, SoundCloud, TikTok, Instagram, Twitter/X, Twitch, and 1,000+ more platforms. Audio in MP3, M4A, WAV, or FLAC. Video up to 4K.
yt-dlp Backend
📂
Local File Processing
Browse any local .mp4, .mkv, .avi, .mov, .webm or audio file. Video files are automatically piped through ffmpeg -vn -q:a 0 -map a to extract audio — no manual steps needed.
ffmpeg Extraction
🔄
Format Conversion
After every processing step, ffmpeg automatically converts the result to your selected format. MP3 via libmp3lame VBR, M4A via AAC 256k, FLAC lossless, WAV PCM copy — whichever you choose in the dropdown.
Auto Convert
🗑️
Auto-Cleanup Pipeline
After vocal isolation, the app auto-deletes the original download, the extracted audio temp file, and the entire htdemucs_ft/ temp folder. Only Song Title (no music).[ext] is kept.
Disk Space Saved
⚡
Extreme Mode & Auto CUDA
Extreme checkbox activates shifts=4 + overlap=0.25 for maximum accuracy. CUDA GPU mode delivers 3–10× speed. If CUDA fails, the app auto-retries on CPU — no user intervention needed.
Smart Retry
🌍
Bilingual UI — Arabic / English
Full bidirectional layout: one click toggles Arabic RTL and English LTR — all labels, placeholders, checkboxes, and combobox entries switch instantly. Preference is persisted via QSettings across launches.
RTL ↔ LTR
🎨
Dark / Light Theme
One-click theme toggle between a deep dark palette and a clean light mode using Qt6 Fusion style. Theme is applied application-wide instantly and persisted across sessions. No restart needed.
Persisted Theme
📦
Offline-First Tool Manager
yt-dlp, ffmpeg, and demucs are downloaded once to ~/.local/share/m4s_d/bin/ (Linux) or %LOCALAPPDATA%\m4s\m4s_d\bin\ (Windows) and reused forever. Click "Setup Tools" to repair.
No Re-Downloads
🔍
Self-Diagnosis on Startup
Every launch checks yt-dlp, ffmpeg, and demucs individually across local dir, pipx, venv, and system PATH. Each result is color-coded in the log. Missing tools print exact fix commands.
3-Tool Check
🔧
torchcodec Audio Fix
Both installers automatically run pipx inject demucs torchcodec, patching the broken torchaudio.backend.sox_io_backend.save() call in modern PyTorch. The app also prints the exact fix command if demucs fails.
Auto-Patched
🚀
Universal Installers
install_linux.sh auto-detects pacman / apt / dnf / zypper, installs all deps, injects torchcodec, compiles the Qt6 app, and creates a .desktop launcher. Windows uses a PowerShell script.
4 Linux Distros + Windows

// processing pipeline

From URL to Perfect Vocals

The pipeline adapts automatically — URL or local file, video or audio, with or without vocal isolation.

0
Input
Paste a URL or browse a local file.

If local file is a video (.mp4, .mkv, etc.), ffmpeg -vn -q:a 0 -map a extracts the audio stream first automatically.
1
Download (URL)
yt-dlp downloads media at your selected quality.

Audio: -f bestaudio -x --audio-format [mp3|m4a|wav|flac]
Video: -f bestvideo[height≤N]+bestaudio --merge-output-format mp4
2
Vocal Isolation
demucs -n htdemucs_ft --two-stems=vocals --shifts=2|4 --overlap=0.1|0.25

Outputs vocals.wav. CUDA auto-fallback to CPU if needed.
3
Convert & Save
ffmpeg converts vocals.wav to the selected audio format.

Saves "Title (no music).[ext]".
Auto-deletes original, temp audio, and demucs folder.

// installation — linux

One Script. Everything Handled.

Detects your distro, installs Qt6 + all tools, compiles the app, creates the .desktop launcher, and injects the torchcodec fix automatically.

bash — terminal
# 1. Clone the repository
$git clone https://github.com/mahmoudelsheikh7/M4S_D.git
$cd M4S_D
# 2. Make executable — do NOT use sudo
$chmod +x install_linux.sh
$./install_linux.sh
# 3. Launch the app
$m4s_d
Note: Do not run with sudo. The script uses sudo internally only where needed.
Install time: ~5–25 min (downloads PyTorch for demucs, ~1–3 GB).
01Detects pacman / apt / dnf / zypper automatically
02Installs Qt6, CMake, build tools, ffmpeg, yt-dlp, pipx
03Installs demucs via pipx install demucs with isolated environment
04Injects torchcodec — fixes audio-save errors in modern PyTorch
05Compiles Qt6 C++17 application from source with CMake
06Installs binary to /usr/local/bin/m4s_d + .desktop launcher

// installation — windows

Windows PowerShell Setup

Automated setup script downloads yt-dlp.exe, ffmpeg.exe, creates a Python venv, and installs demucs with torchcodec.

Step 1 — Run Setup Script
# Allow execution (one-time)
Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned

# Clone and run
git clone https://github.com/mahmoudelsheikh7/M4S_D.git
cd M4S_D
.\install_windows.ps1
Step 2 — Build the Qt6 App (MSYS2 — Recommended)
# In MSYS2 MinGW 64-bit terminal:
pacman -S mingw-w64-x86_64-qt6-base mingw-w64-x86_64-qt6-tools \
          mingw-w64-x86_64-cmake mingw-w64-x86_64-gcc make

cd /c/path/to/M4S_D
mkdir build && cd build
cmake .. -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Release
mingw32-make -j$(nproc)
What the script downloads
✓ yt-dlp.exe — official GitHub release
✓ ffmpeg.exe — GPL static build (~60 MB)
✓ demucs — Python venv + PyTorch
✓ torchcodec — audio-save fix
