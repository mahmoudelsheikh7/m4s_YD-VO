The "No Music" Pipeline

Three automated stages transform any URL into a clean, vocals-only .m4a file.

01
yt-dlp
Download & Extract Audio
Downloads the best available audio stream from the URL and converts it to M4A format at maximum quality.
-f bestaudio[ext=m4a]/bestaudio/best
-x --audio-format m4a --audio-quality 0
→ "Song Title.m4a"
02
demucs · htdemucs_ft
AI Vocal Separation
Meta's htdemucs_ft model with shifts=2 runs source separation to isolate the vocal stem with near-zero music bleed.
-n htdemucs_ft --two-stems=vocals
--shifts=2 -d cpu|cuda
→ outdir/htdemucs_ft/Song Title/vocals.wav
03
ffmpeg
Encode Final Output
Converts the separated vocals WAV to a high-quality 256kbps AAC M4A file ready for playback anywhere.
-c:a aac -b:a 256k
→ "Song Title (no music).m4a"
✓
cleanup
Automatic Cleanup
All intermediate files are removed. Only the clean vocal output remains.
Song Title.m4a
original with music
outdir/htdemucs_ft/
demucs temp folder

Get Up & Running

Pick your platform. The installer handles everything — dependencies, build, and desktop launcher.

bash — ~/M4S_D 
# 1. Clone the repository
$ git clone https://github.com/mahmoudelsheikh7/M4S_D.git
$ cd M4S_D

# 2. Make installer executable
$ chmod +x install_linux.sh

# 3. Run (do NOT use sudo — it calls sudo internally)
$ ./install_linux.sh

The installer automatically handles everything:

install_linux.sh — auto steps 
[1] Detects package manager (pacman / apt / dnf / zypper)
[2] Installs Qt6, CMake, build tools, ffmpeg, yt-dlp, pipx
[3] Installs demucs via pipx install demucs
[4] Injects torchcodec via pipx inject demucs torchcodec
[5] Compiles the Qt6 application with CMake
[6] Installs binary to /usr/local/bin/m4s_d
[7] Creates a .desktop launcher in your application menu

Launch after install:
$ m4s_d
# or open "m4s d" from your application menu

Building on Windows — Option A (MSYS2, Recommended)

MSYS2 MinGW 64-bit 
# Install Qt6 and build tools via pacman
$ pacman -S mingw-w64-x86_64-qt6-base mingw-w64-x86_64-cmake \
           mingw-w64-x86_64-gcc make

$ cd /c/path/to/M4S_D
$ mkdir build && cd build
$ cmake .. -G "MinGW Makefiles" -DCMAKE_BUILD_TYPE=Release
$ mingw32-make -j$(nproc)

 Option B: Install Qt6 from qt.io, open CMakeLists.txt in Qt Creator, and build with Release configuration.

bash — Manual Build (Linux) 
# Arch Linux
$ sudo pacman -S qt6-base qt6-tools cmake base-devel ffmpeg yt-dlp python-pipx

# Ubuntu / Debian
$ sudo apt install qt6-base-dev qt6-tools-dev cmake build-essential ffmpeg pipx python3

# Install demucs with audio-save fix
$ pipx install demucs
$ pipx inject demucs torchcodec

# Build the application
$ mkdir build && cd build
$ cmake .. -DCMAKE_BUILD_TYPE=Release
$ make -j$(nproc)
$ ./m4s_d

Common Issues

Quick fixes for the most frequent setup problems.

demucs: command not found after install
fix: PATH issue 
$ echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc && source ~/.bashrc
Demucs audio-save error (torchaudio.backend.sox_io_backend)
fix: inject torchcodec 
$ pipx inject demucs torchcodec
yt-dlp HTTP 403 error
fix: update yt-dlp 
$ yt-dlp -U   # or: pipx upgrade yt-dlp
App shows all tools as missing on first Windows run
Click ⬇ Setup Tools in the app — it will download and save yt-dlp.exe and ffmpeg.exe to the local data directory automatically. No further action needed on subsequent launches.
